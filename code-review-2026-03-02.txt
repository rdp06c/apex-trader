I. EXECUTIVE SUMMARY
================================================================================

The APEX system is architecturally sound — the two-phase AI decision loop,
multi-layer technical analysis, and self-learning framework are well-designed.
However, the portfolio data tells a clear story: 29% win rate, -3.4% return on
deposits, negative alpha vs SPY (-3.94%). The system is losing money.

The root causes are NOT architectural — they're tuning and enforcement problems:
  1. Entry quality filters are too weak (RS 100 entries dominate, 18% win rate)
  2. Conviction is inflated (almost never below 7, full scale unused)
  3. Cash deployment ignores regime (97% invested during choppy market)
  4. Position sizing is uncapped (NOW = 24.6% of portfolio)
  5. "Opportunity cost" churn destroys value (10 of 31 exits, coin-flip outcomes)
  6. Several derived trading rules are broken or too weak

Code-level bugs found: 26 issues in trader.js, 22 in HTML/CSS/build.
No critical bugs. 7 high-severity issues. 17 medium. Balance is low.

III. CRITICAL & HIGH FINDINGS
================================================================================

[H1] isAnalysisRunning Not Reset on Early Return
  File: src/trader.js, lines ~4465-4494 (runAIAnalysis)
  Category: Bug
  When the zero-data gate fires or user cancels the <70% gate, the function
  returns WITHOUT resetting isAnalysisRunning = false. This permanently locks
  out all future analysis until page refresh.
  FIX: Set isAnalysisRunning = false before every early return, or use finally.

[H2] Pullback Entry Prefer Rule Tests Wrong Field (Dead Code)
  File: src/trader.js, line ~3028 (deriveTradingRules)
  Category: Bug
  The pullback_entry prefer rule filters on entryTechnicals.momentumScore >= -8
  && <= -2. But momentumScore is a 0-10 scale (never negative). This condition
  NEVER matches. The "prefer pullback entries" learning signal is completely dead.
  FIX: Change momentumScore to totalReturn5d (the 5-day return % that CAN be negative).

[H3] XOR "Encryption" Provides Zero Security
  File: src/trader.js, lines ~9937-9958 (encryptKeys/decryptKeys)
  Category: Security
  Uses XOR with GDRIVE_CONFIG.CLIENT_ID as key. Client IDs are public by design.
  Anyone with the Drive file + Client ID can decrypt API keys trivially.
  FIX: Use Web Crypto API (AES-GCM) with user-provided password, or document
  this as obfuscation-only (acceptable given drive.file scope).

[H4] External CDN Scripts Without Subresource Integrity (SRI)
  File: src/template.html, lines 10, 12
  Category: Security
  Chart.js loaded from jsDelivr without integrity hash. CDN compromise = full
  access to localStorage (API keys, portfolio data).
  FIX: Add integrity="sha384-..." crossorigin="anonymous" to Chart.js script tag.

[H5] Accessibility: Interactive Divs Without Keyboard Support (4 patterns)
  Files: src/body.html line 11, src/trader.js (generated HTML)
  Category: Accessibility
  cloudSyncStatus, holding-card-catalyst, decision-stock-item, and
  decision-thoughts/research-summary all use <div onclick> without role="button",
  tabindex="0", or keyboard event handlers. Keyboard users cannot interact.
  FIX: Add role="button" tabindex="0" onkeydown handler to all interactive divs.

[H6] RS 100 Penalty Too Weak — Entries Still Dominating
  File: src/trader.js (composite scoring)
  Category: Trading Logic
  RS >= 95 gets -3 penalty, but 11 of 22 tracked trades entered at RS 100 with
  only 18% win rate. The penalty is NOT preventing these entries.
  FIX: Double the penalty (RS >= 95 -> -6) or make it a block rule in deriveTradingRules.

[H7] Overbought RSI Rule Should Be Block, Not Warn
  File: src/trader.js (deriveTradingRules)
  Category: Trading Logic
  RSI >70 at entry: 17% win rate, avg -4.12%. Currently only "warn" level.
  6 trades is enough data to confirm this pattern.
  FIX: Upgrade overbought_rsi from "warn" to "block" enforcement level.

IV. MEDIUM FINDINGS
================================================================================

[M1] Migration Flags Immediately Deleted (Runs Every Load)
  File: src/trader.js, lines ~7862-7884 (loadPortfolio)
  _exitReasonV2 flag set then deleted. Same for _scoringV2. Migrations re-run
  on every page load. Idempotent but wasteful.
  FIX: Remove the delete lines so flags persist.

[M2] fetchGroupedDailyBars Doesn't Account for US Holidays
  File: src/trader.js, lines ~1282-1295
  Skips weekends but not holidays. ~10 holidays/year means ~38 bars instead of 40.
  In holiday clusters (Thanksgiving week), could get notably fewer bars.
  FIX: After fetching, if <35 actual trading days, extend date range.

[M3] fetchGroupedDailyBars Today-Bar Dedup Uses Local Timezone
  File: src/trader.js, line ~1372
  Uses browser local timezone for date comparison instead of ET. Users in
  timezones ahead of Eastern could get duplicate or missing bars.
  FIX: Use ET timezone comparison like isMarketOpen() does.

[M4] uploadDecisionToDrive Doesn't Use gdriveApiFetch (No Token Refresh)
  File: src/trader.js, lines ~8667-8671, ~8742-8748, ~9997-10044
  Several Google Drive functions use raw fetch() instead of gdriveApiFetch()
  which handles 401 token refresh. Mid-session token expiry = silent failures.
  FIX: Use gdriveApiFetch() consistently for all Drive API calls.

[M5] portfolio.transactions Grows Unbounded
  File: src/trader.js, lines ~7118, 7330
  No cap on transactions array. Over months, could hit localStorage 5-10MB limit.
  FIX: Add rolling cap (e.g., last 500 transactions, summarize older).

[M6] Phase 2 Candidate Count Can Overwhelm Context Window
  Per CLAUDE.md: no hard cap on candidate count, can exceed 50+.
  FIX: Hard cap at 35-40 candidates, summarize the rest.

[M7] Feedback Loop Risk in Signal Accuracy Adjustments
  File: src/trader.js, lines ~2023-2037 (getSignalAccuracyAdjustments)
  Learned penalties reduce trades with that signal, which reduces recovery data
  points, which locks in the penalty. No decay or recency weighting.
  FIX: Weight recent trades more heavily, or cap adjustments at +/-1.

[M8] Encryption Password Falls Back to Hardcoded Default
  File: src/trader.js, lines ~10159-10163 (getEncryptionPassword)
  If CLIENT_ID is empty, password = 'apex_default_key_2026' (public string).
  FIX: Refuse to encrypt/decrypt without a real Client ID.

[M9] Partial Sell Records Wrong Hold Time
  File: src/trader.js, lines ~7222-7327 (executeSingleTrade)
  Hold time uses first buy's timestamp, ignoring subsequent add-to-positions.
  FIX: Weight hold time by shares from each buy transaction.

[M10] max-height: 5000px Transition Hack for Collapsible Sections
  File: src/styles.css, line ~444
  Expanding sections animate through the full 0-5000px range causing timing
  mismatch. Collapsing feels normal, expanding has visible delay.
  FIX: Use JS to measure scrollHeight and set max-height dynamically.

[M11] Global button Styles Leak Into All Button Elements
  File: src/styles.css, lines ~770-793
  Base button selector applies margins/padding to every <button>, forcing
  overrides throughout the CSS.
  FIX: Scope to .sidebar-actions button or add .btn-primary class.

[M12] activity-feed max-height Conflicts With Collapsible Pattern
  File: src/styles.css, lines ~688-691
  overflow-y:auto on the outer card clips the collapsible animation.
  FIX: Move max-height to inner #activityFeed div.

[M13] Chart.js Loaded Synchronously (Render-Blocking)
  File: src/template.html, line 10
  200KB+ library blocks HTML parsing.
  FIX: Add defer attribute to the script tag.

[M14] No Content Security Policy (CSP)
  File: src/template.html
  No CSP meta tag. Inline onclick handlers require 'unsafe-inline' anyway,
  but even a permissive CSP restricts script sources.
  FIX: Add <meta http-equiv="Content-Security-Policy"> with domain whitelist.

[M15] aria-expanded Not Dynamically Updated
  File: src/body.html, lines 65, 288, 306, etc.
  Section headers have static aria-expanded. If toggleSection() doesn't update
  this attribute, screen readers won't know current state.
  FIX: Verify toggleSection() updates aria-expanded on toggle.

[M16] Modal Lacks Focus Trap and aria-modal
  File: src/body.html, lines ~400-406
  #resultModal has no role="dialog", aria-modal, or focus trap.
  FIX: Add ARIA attributes and implement focus trap in JS.

[M17] Cash Deployment Ignoring Regime
  Category: Trading Logic
  System is 97% invested during choppy regime with 29% win rate. Phase 2 has
  regime-based cash deployment guidance but it's clearly not being followed.
  FIX: Add code-enforced minimum cash reserve per regime (e.g., choppy = 20% min cash).

VI. ARCHITECTURE RECOMMENDATIONS
================================================================================

1. STRUCTURED OUTPUT FOR AI RESPONSES
   The 5-layer JSON parsing recovery system is impressive but symptoms of a
   fundamental problem. Consider using Anthropic's tool_use response format for
   Phase 1 and Phase 2, which guarantees valid JSON structure and eliminates
   most parsing fallbacks.

2. EXTRACT runAIAnalysis INTO SUB-FUNCTIONS
   At ~2,460 lines, this function is the single biggest maintainability risk.
   Extract: buildPhase1Prompt(), parsePhase1Response(), buildPhase2Prompt(),
   parsePhase2Response(), preparePhase2Candidates(). This is a significant
   refactor — discuss before implementing.

3. CODE-ENFORCED TRADING GUARDRAILS
   The derived trading rules system is good but several rules are too weak or
   broken. Consider adding hard-coded guardrails that can't be overridden:
     - Max position size: 15% of portfolio
     - Min cash reserve: 10% (20% in choppy/bear regime)
     - Block RSI >70 entries (or require exceptional other signals)
     - Block RS 100 entries (mean-reversion trap)
     - Block entries during sector "caution" flow

4. WINNER MANAGEMENT
   CCI was sold at +2.82% and then ran +11.45%. The system needs better "let
   winners run" logic:
     - Trailing stop based on ATR or structure (not fixed %)
     - Phase 1 prompt guidance: "Don't sell a stock that's still in bullish
       structure just because something else scores higher"
     - Track opportunity cost of sells (already done) and inject this as
       learning signal

5. CONVICTION CALIBRATION
   The AI uses 7-10 for nearly everything. Add prompt guidance:
     - "Use the FULL 1-10 scale. Conviction 5-6 should be common for speculative
       entries. 9-10 should be rare (<=20% of buys)."
     - Or normalize AI-provided conviction scores programmatically.

6. SECTOR-AWARE POSITION SIZING
   Technology is -$134.88 over 11 trades. The system should:
     - Track sector win rates in deriveTradingRules
     - Reduce position size for sectors with <33% win rate
     - Or block new entries in sectors with sustained losses
